General Information


Page Header

All the pages in the application use a standard page header that includes:

  • At the top: Links to the Home page, Contact Us page and this Help page
  • Next to the application title: Your Login and User Names are displayed, as well as the User Group to which you belong
  • At the top: A series of links showing where you are in the application and allowing you to navigate back to a previous page

Required Fields

Required fields will be marked with a * at the end of the field name. These fields must be entered in order to insert or update a record in the repository.

Calendar Controls

In a number of places you will see the following icon displayed next to date fields.

  • By clicking the icon a calendar will be displayed allowing you to select the relevant date
  • To close the calendar without selecting a date, click the icon again
  • Please try and use the calendars when entering date fields as opposed to manual entry to help avoid formatting problems

Picklists

Where the fields are limited to a picklist of values, these can behave in two ways.

  • For long lists, if you click the field and start typing, the list will jump to the first matching entry of the list. This is shown by "Type to search" appearing above the list.
  • For short lists, the list will jump to the entries starting with the last character typed.

Validation Messages

Validation messages will be displayed when data is either entered in correctly or omitted, and are displayed in one of two ways:

  • Beneath fields: In most cases an instruction written in red text will appear beneath the field instructing you as to what is required, these will usually vanish once corrected
  • At the bottom of the page: Some validation messages will only appear at the bottom of the page, these will only disappear when you click one of the links again

Home Page

This is the Home Page, it acts as a central switchboard, allowing you access to various aspects of the application.

To start using the application you will need to click the one the links displayed:

  • Register or Update Submission, Samples and Isolates: Allows you to add, update and delete submission, sample and isolate records from the repository
  • Search Repository: Allows you to search for records with the repository
  • Relocate Virus Isolates: Allows you to relocate virus isolates
  • Produce Reports: Allows you to produce reports
  • Maintain Look-up Data: Allows you to maintain the look-up lists used by the repository
  • Maintain Virus Characteristics: Allows you to maintain the list of virus characteristics used by the repository
  • Maintain Sender Information: Allows you to maintain the list of senders used by the repository
  • View Audit Logs: Allows you to view the audit logs
  • System Information: Allows you to view the system information
  • : Opens a new window displaying the User Management application. Allows you to manage user access to the application

Note: Some of these options will only be displayed if you are a part of the relevant User Group

Submission Selection Page

In order to insert or update a submission, or it's components you will first need to specify it's AV or PD number:

  • Enter the AV, PD, SI or BN number into the space provided
  • AV numbers should be in the format AV000000-00
  • PD numbers should be in the format PD0000-00
  • SI numbers should be in the format SI000000-00
  • BN numbers should be in the format BN000000-00

Click the link or press the [Enter] key to continue:

  • New Submission: You will be redirected to the Submission Details Page
  • Existing Submission: You will be redirected to the Submission Summary Page

Senders Details

The main senders are held in a list for quickly adding their details to the submission.

  • Clicking the 'Main Senders' link brings up a blue panel with a drop down list of senders, sorted by sender name, organisation and country.
  • Clicking the 'Main Organisations' link brings up a blue panel with the same list, but sorted by organisation, sender name and country.
  • These lists are limited to those senders from the country in the submitting country field, so if you can't see the one you want, clear this field.

Search Repository Page

This page allows you to search the repository for isolates matching the criteria specified.

Search Criteria

Select values from the picklists that are going to be included in the search:

  • AV Number: The AV Number of the submission
  • Virus Family: The virus family
  • Virus Type: The virus type
  • Virus Characteristics: The virus characteristics, limited to those available for the selected virus type.  Up to three characteristics can be selected.

When you select a Virus Characteristic, the system will offer you operators and entry fields/pick list appropriate to the Characteristic type:

List Type:
e.g. 'H Type' 		= if you don't select a value from the picklist, the search will look for isolates with that characteristic type (whether they have a value or not) and return them all
if you select a value from the pick list, the search will look for isolates that have that characteristic set to the value you selected in the pick list
not equal to if you don't select a value from the pick list, the search will look for isolates with that characteristic type that have a value entered (blank entries will not be returned)
if you select a value from the pick list, the search will look for isolates that have that characteristic and a value not equal to the one you selected (blank entries will be returned)
begins with if you don't enter a value in the field, the search will look for isolates with that characteristic type (whether they have a value or not) and return them all
if you enter a value in the field, the search will look for isolates that have that characteristic and have a value that begins with the value you have entered
Numeric Type:
e.g. 'IVPI'		 = if you don't enter a value in the field, the search will look for isolates with that characteristic type (whether they have a value or not) and return them all
<, >, <=, >= if you select one of these comparators, the system will compare the value you enter in the field with the value held against that characteristic and return all isolates that satisfy the comparison
between you can enter two values between which (inclusively) you want to search for - if you leave any of the two fields blank it will in effect perform either a >= or <= comparison depending on which of the two you left blank - for example, if you put 0.5 in the first field and leave the second field blank, the system will search for values >=0.5
Yes/No Type:
e.g. 'Calcification Present'		 = select either 'Yes' or 'No' from the pick list
Text Type:
e.g. 'Descriptive Analysis'		 = enter the text you wish to search for in the field - note: only those isolates that match that text exactly will be returned
contains enter the text you wish to search for in the field - the system will return isolates that have that text anywhere in the characteristic value
  • Group: The host group
  • Species: The host species
  • Country of Origin: The country of origin
  • Year of Isolation: The year of isolation
  • Year Submission Received: The year of submission
  • Received From Date: The minimum received date
  • Received To Date: The maximum received date

Once you have finished entering criteria click the link.

The system will return isolates matching ALL of the criteria you specified
